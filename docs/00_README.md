# todly 문서 패키지 (Index)

todly 디자인(`Tudly.dc.html`)을 분석하여 기획·설계·구현 프롬프트를 한 세트로 정리한 문서입니다.
화면 디자인은 **Claude Design 내보내기 HTML(`Tudly.dc.html`)을 단일 진실 소스**로 사용하며,
**스크린샷/이미지는 사용하지 않습니다** — Claude Code가 HTML을 직접 읽어 React로 옮깁니다.

> **v2.0 (실제 디자인 전면 반영)**: 초기 문서는 PDF 4개 화면 기준이라 일부를 추론했으나,
> 실제 `Tudly.dc.html`에는 **완성 화면 19개**가 있어 전면 교정했습니다. 주요 변경:
> ① 화면 4개+추론 → **19개 전부 확인** ② **디자인 토큰 실측 교정**(Primary `#1366CE`/`#2E86E6`,
> 프로필 4색 `#2E86E6/#2BC4B0/#FF9D52/#6B5BD0`, 상태색 `#FF7A6B/#FFB23E/#46D38A`, 폰트 Pretendard+**Sora**)
> ③ 누락 기능 정식화: **친구 시스템 · 라이브 룸(사진·응원) · 게이미피케이션(점수·잔디) · 사진/댓글 · 5테마/다크모드 · "내가 할게요" · 그룹 타입 · @아이디**.

## Claude Design 연동 (권장)
HTML을 매번 파일로 두지 않고 디자인 프로젝트에서 직접 동기화하려면 **claude_design MCP**를 연결합니다.
- 연결 명령: **`/design-login`** (권한 `user:design:read/write` 추가)
- 디자인 프로젝트: `https://claude.ai/design/p/724cdab2-e093-41dd-90b1-3af4900723b8?file=Tudly.dc.html`
- 미연결 시에는 `Tudly.dc.html` 파일을 저장소 `design/`에 두고 진행하면 됩니다.

## 빠른 시작
1. `Tudly.dc.html`을 저장소 `design/`에, `01`~`04`를 `docs/`에 둔다.
2. `05_빌드_프롬프트.md`의 **[MASTER PROMPT]**를 Claude Code에 붙여넣는다.
3. `.claude/agents/`에 **서브에이전트 정의**(05 문서 내 스니펫)를 저장한다.
4. **PHASE 0**(HTML 분석·토큰 추출)부터 순서대로 진행. 빠른 데모는 **MVP 단축 프롬프트** 사용.

## 디자인에서 확인된 핵심 차별점 (19개 화면)
- **공유 투두**: 그룹/커플/여행/리스트 타입의 공동 목표
- **그 순간 함께하기**: "내가 할게요" → 라이브 시작 → **라이브 룸**(다인원·사진 공유·이모지 응원)
- **꾸준함 게이미피케이션**: 라이프/루틴 점수 · 연속일수 · **잔디 heatmap**(16주/루틴별)
- **친구 그래프**: @아이디로 친구 맺고 그룹에 초대, 온라인 상태
- **능동적 챙김 + 진행률**: "확인이 필요해요", "N개 중 M개 완료" 막대 + 멤버 아바타
- **개인화**: 5테마(오션/민트/바이올렛/코랄/선셋) + 다크모드
